大家好，今天给大家分享 Redback Portal 的设计系统。这个系统定义了整个门户的视觉语言，从色彩、排版到组件样式和响应式策略，确保所有页面保持一致的品牌形象和用户体验。

01 · Visual Theme

视觉主题与氛围

现代能源科技美学 — 在 SaaS 简约与行业权威之间取得平衡

品牌核心

高对比黑白界面，唯一的签名橙 #FF5B01 作为品牌锚点和交互强调色。温暖、饱和的橙色传递活力与亲和力。

排版系统

Roboto Regular 为主 UI 字体，HarmonyOS Sans SC 支持中文。流体排版 clamp() 实现从 360px 到 2560px+ 的无缝缩放。

动画节奏

GSAP 滚动触发入场动画，统一 0.3s ease-in-out 过渡节奏。y: 50 → 0, opacity: 0 → 1 为标准入场模式。

结构模式 · 固定白色头部 → 全屏 Hero → 交替黑白内容块 → 黑色页脚

设计哲学 · 内容驱动布局，颜色和间距定义层级，而非模拟物理深度

这是整个设计系统的总览。核心是单一强调色策略 — 只有 #FF5B01 这一个橙色作为品牌信号。排版采用流体 clamp() 缩放，动画节奏统一在 0.3 秒。整体结构是白色头部、全屏视频 Hero、交替黑白内容区和黑色页脚。

流体排版是这套系统的核心亮点。通过 clamp() 函数，文字尺寸在最小值和最大值之间按视口宽度比例缩放，完全不需要断点覆盖。从 10px 到 80px，10 个层级覆盖所有场景。这意味着设计师和开发者不需要为每个断点单独设置字号。

深度系统极其克制。整个界面几乎不用阴影——只有语言选择器下拉用 shadow-lg。层级通过 z-index 分层和背景色对比来表达。黑色页脚通过极致的色彩对比形成视觉锚点，橙色移动端头部通过色彩主导建立品牌。文字阴影只有一种：1px 1px 0px rgba(0,0,0,0.2)，用于覆盖层文字可读性。

响应式策略

断点宽度头部Hero布局内边距

Laptop ≥ 1200px 白色 · 水平导航 全屏视频 双列开始 60px

移动端特殊处理

表单 16px 防 iOS 缩放 · CTA 圆角 4px · padding 更紧凑

触控目标

CTA min-h 48px · 移动 min-w 160px · 图标 24×24px

折叠策略

Hover 下拉 → 手风琴 · 双栏 → 单列 · 输入+按钮并排 → 堆叠

响应式系统有四个断点：1024px 以下是移动端，1200px 笔记本，1440px 标准桌面，2560px 超宽。移动端和桌面端是完全不同的体验——橙色头部 vs 白色头部、汉堡菜单 vs 水平导航、静态图片 vs 视频背景。触控目标至少 48px 高，表单输入在移动端用 16px 字号防止 iOS 自动缩放。

08 · Guidelines

设计规范 — Do's & Don'ts

✓ Do — 遵循规范

  • 使用 #FF5B01 作为唯一 CTA 和强调色
  • 流体排版 clamp() 处理响应式字号
  • 统一 transition-all 0.3s 过渡节奏
  • 桌面全屏视频 Hero + 移动静态图片回退
  • 移动优先，1024px 断点为移动/桌面分界
  • 所有图片使用 object-cover 保持比例
  • GSAP 入场动画 y:50→0, opacity:0→1
  • 黑底白字作为标准页尾模式

✗ Don't — 禁止事项

  • 不要使用 #FF5B01 以外的颜色作为主 CTA
  • 不要给卡片或内容区添加阴影
  • 不要使用 Regular/Medium/Bold 之外的字重
  • 不要调整 letter-spacing
  • 不要让移动端头部使用白色背景
  • 不要在 UI 元素上使用渐变
  • 不要超出 1440px max-width 而不扩展断点
  • 不要在 CTA hover 使用 scale 变换
这是整个设计系统的红线。最重要的原则：只用一个强调色 #FF5B01、不用阴影用对比表达层级、三种字重就够了、不调 letter-spacing。移动端头部必须是橙色，CTA hover 只变色不变大小。这些规则确保整个门户保持一致的品牌形象。

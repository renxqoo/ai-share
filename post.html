构建一致的 Design System：从颜色到组件

W

Wang Renren

Frontend Engineer

在现代 Web 开发中，设计系统已经从"锦上添花"变成了"不可或缺"的基础设施。一个成熟的设计系统不仅能让团队协作更高效，更能确保产品在不同页面和场景中保持一致的视觉语言和用户体验。

为什么需要设计系统？

当我们开始构建 Redback Technologies 的门户站点时，面临的最大挑战不是技术选型，而是如何让 20+ 页面看起来像是同一个产品。没有设计系统的情况下，每个开发者对"标题应该多大"、"按钮应该是什么颜色"都有自己的理解。

好的设计系统不是限制创意，而是让创意在一致的框架内自由发挥。

色彩系统：单一强调色策略

我们选择了 #FF5B01（Redback Orange）作为唯一的强调色。这个决定在当时引发了讨论——"为什么不能用蓝色做辅助链接色？"

答案是：单一强调色策略让用户的注意力更集中。当页面上只有一个颜色会"跳出来"时，用户的视觉路径是可预测的——从标题到正文到 CTA，形成自然的阅读流。

色彩角色定义

  • 品牌强调：CTA、激活状态、品牌高亮 — #FF5B01
  • 主文本：标题、正文 — #201D1B（暖黑，比纯黑更柔和）
  • 辅助文本：描述、版权 — #A4A9B3（冷灰，不抢注意力）
  • 边框分隔：容器边界 — #E5E6EB

流体排版：告别断点覆盖

传统做法是为每个断点定义不同的字号：text-sm md:text-base lg:text-lg。这不仅冗余，而且在断点切换处会有突变。

我们使用 clamp() 函数实现流体排版：

/* 从 14px 到 16px，按视口宽度线性缩放 */
.text-base-fluid {
  font-size: clamp(0.875rem, 1.11vw, 1rem);
  line-height: 1.5;
}

/* 从 48px 到 64px 的大标题 */
.text-5xl-fluid {
  font-size: clamp(3rem, 4.44vw, 4rem);
  line-height: 1.1;
}

组件规范：按钮系统

按钮是用户交互的核心。我们定义了三类按钮，每类都有严格的规格：

  • Primary Action：日常操作按钮，14px medium 字重，py-2 px-6，圆角 6px
  • Motion CTA：主转化按钮（"Get A Quote"），18px，hover 时文字左移+箭头滑入
  • Newsletter Submit：页脚订阅按钮，与输入框紧贴，深色背景上使用

响应式策略：1024px 分界

我们没有选择多断点渐进增强，而是在 1024px 做了一个硬切分——移动端和桌面端是完全不同的体验。

移动端头部是全橙色背景（品牌识别优先），桌面端是白色水平导航（内容优先）。这不是偷懒，而是承认移动端和桌面端用户的使用场景本质上不同。

总结

一套好的设计系统应该像空气一样——你感觉不到它的存在，但离开它一切都无法运作。从颜色、排版到组件，每一个决策都应该是有原因的、可解释的、可复用的。

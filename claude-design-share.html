为什么不能一上来就让 AI 直接写？

Agent Loop
10 / 18

Agent 是怎么一轮轮工作的？

pseudo.tswhile (!taskComplete) { const response = LLM(messages, tools); // 判断下一步 if (response.toolCalls) { const results = executeTools(response.toolCalls); messages.push(...results); // 结果回到上下文 } else { return response.content; // 输出最终答案 } }
读取当前上下文
调用 LLM 判断下一步
Decision
是否需要调用工具？
是
执行工具，再把结果放回上下文
否
直接返回最终结果
Agent 不是一次性把答案想完，而是带着最新上下文一轮轮继续判断和执行。

assistant调用工具 getWeather({ city: "杭州" })

下一轮不是重新开始，而是把上一轮的 tool call 和工具结果一起带回给 LLM。
Tool Calling
12 / 18

LLM 为什么不只是会回答，还能调用工具？

tools.tsconst tools = { getWeather: { description: "查询指定城市的实时天气", execute: ({ city }) => { return `${city}，晴，25°C，东北风 2 级`; } } }; const call = { name: "getWeather", args: { city: "杭州" } }; const result = tools[call.name].execute(call.args);
类比理解
可以把 tools 理解成一张菜单。
tools：菜单上有哪些服务可以点
name：点的是哪一项服务
args：这次具体要什么
execute()：后台真正去完成服务
result：最后返回给模型的结果
LLM 更像是在下单，不是在亲自做事。
LLM 负责决定“调用什么”，外层程序负责真正“把事情做完”。

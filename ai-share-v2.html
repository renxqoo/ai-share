为什么不能让 AI 一上来就写代码？

1
代码结构很快失控一上来就写，缺少整体规划
2
返工成本高需要反复纠偏，来回修改
3
代码质量差功能能跑，但架构混乱
4
维护成本高最后还得人工重构

最大的问题不是写不出来，而是写出来以后越来越难收拾。

AI Agent 实现原理

为什么 AI 能帮你干活？

LLM 大模型 + TOOLS 工具 + LOOP 循环执行 = AI Agent

它看起来像在"自己干活"，本质上是因为它会结合上下文反复判断，并在需要时调用工具。

Agent 循环

flowchart TD A["接收任务\n用户说一句话"] --> B["LLM 思考\n这件事要怎么做"] B --> C{"LLM 决定"} C -- "自己能回答" --> D["直接输出结果"] C -- "需要工具" --> E["决定调用哪个工具\n传什么参数"] E --> F["系统执行工具\n阅读文档 / 访问浏览器 / 写文件"] F --> G["把工具结果返回给 LLM"] G --> B style A fill:#faf9f5,stroke:#cc785c,stroke-width:2px,color:#141413 style B fill:#faf9f5,stroke:#cc785c,stroke-width:2px,color:#141413 style C fill:#efe9de,stroke:#cc785c,stroke-width:2px,color:#141413 style D fill:#faf9f5,stroke:#5db8a6,stroke-width:2px,color:#141413 style E fill:#faf9f5,stroke:#e8a55a,stroke-width:2px,color:#141413 style F fill:#faf9f5,stroke:#cc785c,stroke-width:2px,color:#141413 style G fill:#faf9f5,stroke:#cc785c,stroke-width:2px,color:#141413

Agent 的本质不是一次生成答案，而是"判断 → 调用工具 → 看结果 → 再判断"的循环。

Agent 的执行机制

while (!taskComplete) {
  // 基于当前上下文和可用工具，判断下一步
  const response = LLM(messages, tools);

  // 如果模型认为需要调用工具
  if (response.toolCalls) {
    // 系统执行工具调用
    const results = executeTools(response.toolCalls);

    // 把工具结果放回上下文
    messages.push(...results);
  } else {
    // 不需要工具，直接输出
    return response.content;
  }
}
第 1 轮

用户：帮我查一下杭州今天的天气

LLM 判断：这是实时信息，不能直接编

动作：调用天气工具

第 2 轮

工具返回：杭州，晴，25°C，东北风 2 级

LLM 判断：信息已经够了

动作：整理结果，准备回复

第 3 轮

输出最终答案，循环结束

Agent 不是一次想完答案，而是带着最新上下文反复判断和执行。

一句话理解 Agent

Agent 不是魔法，
它只是把"会回答"变成了"会行动"。

带着上下文反复调用模型 模型决定要不要用工具 工具结果回来后再继续下一轮

我们能不能给 Agent 一套更好用的操作方式？

既然 AI Agent 已经能处理信息和任务

那我们公司的项目，能不能也提供一种
更适合 Agent 操作的方式？

不是重做系统 · 而是换一种更适合 Agent 操作的方式

思路：把业务操作变成 CLI 命令

# 查看待处理的付款单
saas payment list --status=pending
# 查看某笔付款详情
saas payment detail --id=PAY20260509001
# 查询实时汇率
saas deal quote --pair=USD/CNH
# 查看待审批列表
saas approval list --pending
# 查询余额
saas balance check --currency=USD

CLI 不是重做一套系统，而是给现有系统再加一层适合 Agent 调用的壳。

Agent 怎么接手这些业务动作？

你：查一下今天有多少笔待审批的付款

Agent：执行 saas approval list --pending

当前有 12 笔待审批付款，其中 3 笔超过 24 小时未处理

你：把金额最大的那笔详情给我看

Agent：执行 saas payment detail --id=PAY20260509087

金额：USD 500,000，状态：待审批

你：这笔交易的汇率是多少？

Agent：执行 saas deal quote --pair=USD/CNH

当前 USD/CNH 汇率：7.2450

自然语言 → Agent → CLI → API → 结果返回

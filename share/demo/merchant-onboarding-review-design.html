背景

设计目的

本文档定义了在 /Users/wrr/work/forex-kybc broker/admin 项目中支持 Merchant 入驻审核的前端实现方案。

核心规则

Merchant 与 FX 必须通过 category 分流

Merchant 与 FX 的分支判断必须由列表页/详情页路由参数 category 决定。

不要使用 bizType 来区分 Merchant 与 FX。

bizType 应继续表示现有 KYX 业务类型区分，例如 KYC / KYB / ID。

路由规则

路由参数策略

列表页

onboarding 列表页必须将 category 透传到详情路由。

/broker/onboarding-list/onboarding-detail?id=32820&queryType=ONBOARDING&type=Organization&bizType=2&category=Merchant

详情页

详情页必须优先从路由参数中读取 category。

const category =
  searchParams.get('category') ||
  searchParams.get('source') ||
  kyxCommonInfo?.source;
  • category 是新的唯一真实来源参数。
  • source 仅作为临时兼容兜底保留。
  • 如果 category === 'Merchant'，必须进入 Merchant 流程。
  • 否则默认保持现有 FX 兼容行为。

Tab 行为

FX 与 Merchant 对比

Tab FX Merchant
Details 无行为变更，保留现有 FX 渲染逻辑。 使用 Merchant 专属详情渲染器。不要复用现有 FX BusinessInfo 组合，因为它依赖 businessDetails, registeredOfficeAddress, principalBusinessAddress, mailingAddress, contactDetail, additionalInfo, and wholesales。

Documents 无行为变更。 Merchant 也保持不变。

注册号信息区块

不要将标签硬编码为 ABN。

详情 API 可能返回 ABN、ACN 或其他注册号类型。

UI 应根据 registrationInfoList[i].type 渲染。

  • 如果 registrationInfoList 有数据，则逐项渲染。
  • label = item.type
  • value = item.number

  • 当 businessModel === 'Online' 时展示
  • 当 businessModel === 'Both' 时展示
  • 当 website 已有非空值时展示

Operating Address

  • 当 businessModel === 'Offline' 时展示
  • 当 businessModel === 'Both' 时展示
  • 当 operatingAddress 中已经存在可见值时展示

展示规则

空值与地址格式化

空值规则

Merchant Business Information 中所有空值都应显示为：

Not filled
  • 缺失的标量字段
  • 地址子字段为空
  • 后端返回的 null 字符串值

地址格式化规则

  • 对于 Registered Address 和 Operating Address 的摘要行，若存在 address，优先使用它。
  • 否则由 streetNumber、streetName、suburb、state、postcode、country 组合为可读字符串。
  • 如果所有字段都为空，则显示 Not filled。

验证清单

需要确认的内容

路由

  • onboarding 列表详情链接包含 category
  • 详情页能正确读取 category
  • Merchant 数据进入 Merchant 分支
  • 非 Merchant 数据仍然保持 FX 兼容分支

Details Tab

  • Merchant 申请信息渲染正确
  • Merchant 注册号类型能按 API 返回值正确展示
  • Merchant 业务信息字段按预期顺序展示
  • 地址摘要行展示正确
  • 空值显示为 Not filled
  • Website URL 的可见性符合 business model 规则
  • Operating Address 的可见性符合 business model 规则

Review Tab

  • Merchant review 不渲染 KybCheckSummary
  • Merchant review 不渲染 KycCheckSummary
  • Merchant review 不渲染 Reports
  • FX review 保持不变

Users Tab

  • Merchant 用户列表不渲染 Action 列
  • FX 用户列表保持不变

Documents Tab

  • documents 展示与上传行为无回归

风险

需要关注的点

  • category 后续可能扩展出更多 source 值
  • registrationInfoList 可能包含多项
  • Merchant 详情 API 在 users/documents/review 上可能返回大量 null 值
  • 枚举值必须始终与 Merchant C 端实现保持一致

非目标

本设计不定义的内容

  • 后端 schema 变更
  • source 与 category 的 API 命名统一
  • 对现有 FX 组件做深度重设计

最终决策摘要

实现默认方案

  • source 分支判断使用 category
  • Merchant 详情数据来源为 businessInfoVO.merchantBusinessDetails
  • Merchant Details 使用独立渲染器
  • Merchant Review 移除检查摘要与 reports 模块
  • Merchant Users 移除 action 列
  • Documents 保持不变

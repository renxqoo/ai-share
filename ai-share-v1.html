Why Not

为什么不能让 AI 一上来就写代码？

01 结构失控
一上来就写，代码结构很快失控。

它能补出功能，却不一定知道该落在哪一层、复用哪一套约定。

02 返工成本高
需要反复纠偏，返工会把节省的时间吃回去。

需求没对齐时，生成越快，偏航也越快。

03 质量不稳
功能能跑，但代码质量、边界和安全性可能都不够。

“看起来对”不等于“适合长期维护”。

04 维护变难
最后还是要人工重构，后续维护成本更高。

真正麻烦的不是写不出来，而是写出来以后越来越难收拾。

最大的问题不是写不出来，而是写出来以后越来越难收拾。

Agent Loop

Agent 循环

flowchart TD A["接收任务
用户说一句话"] --> B["LLM 思考
这件事要怎么做"] B --> C{"LLM 决定"} C -- "自己能回答" --> D["直接输出结果"] C -- "需要工具" --> E["决定调用哪个工具
传什么参数"] E --> F["系统执行工具

Agent 的本质不是一次生成答案，而是“判断 → 调用工具 → 看结果 → 再判断”的循环。
Execution Mechanism

Agent 的执行机制

Pseudo Code
while (!taskComplete) { // 基于当前上下文和可用工具判断下一步 const response = LLM(messages, tools); if (response.toolCalls) { // 系统执行工具调用 const results = executeTools(response.toolCalls); // 工具结果进入下一轮上下文 messages.push(...results); } else { // 不需要工具时输出最终结果 return response.content; } }
第 1 轮
用户：帮我查一下杭州今天的天气
LLM 判断：这是实时信息，不能直接编
动作：调用天气工具
第 2 轮
工具返回：杭州，晴，25°C，东北风 2 级
LLM 判断：信息已经够了
动作：整理结果，准备回复
第 3 轮
输出最终答案
循环结束
Agent 不是一次想完答案，而是带着最新上下文反复判断和执行。
Context & Tools

messages 和 tools 是什么？

messages
[ { "role": "system", "content": "你是个人 AI 助手，需要理解用户的问题，并结合上下文和工具能力帮助用户解决问题。" }, { "role": "user", "content": "帮我查一下杭州今天的天气" }, { "role": "assistant", "tool_calls": [ { "id": "call_1", "function": { "name": "get_weather", "arguments": "{\"city\":\"杭州\"}" } } ] }, { "role": "tool", "content": "杭州，晴，25°C，东北风 2 级" } ]
tools
[ { "type": "function", "function": { "name": "get_weather", "description": "查询指定城市的天气信息", "parameters": { "type": "object", "properties": { "city": { "type": "string", "description": "城市名称，例如：杭州" } }, "required": ["city"] } } } ]
messages 决定它“看到了什么”，tools 决定它“能做什么”。

Business Direction

我们能不能给 Agent 一套更好用的操作方式？

核心问题
既然 AI Agent 已经能处理信息和任务， 那我们公司的项目，能不能也提供一种更适合 Agent 操作的方式？
这就是后面这一部分想展开的方向。
不是重做系统，而是换一种更适合 Agent 操作的方式。
CLI Layer

思路：把业务操作变成 CLI 命令

Business CLI
$ saas payment list --status=pending $ saas payment detail --id=PAY20260509001 $ saas deal quote --pair=USD/CNH $ saas approval list --pending $ saas balance check --currency=USD
CLI 不是重做一套系统，而是给现有系统再加一层适合 Agent 调用的壳。
Business Agent Flow

Agent 怎么接手这些业务动作？

Round 1
你：查一下今天有多少笔待审批的付款
Agent：执行 saas approval list --pending

Round 2

你：这笔交易的汇率是多少？
Agent：执行 saas deal quote --pair=USD/CNH
